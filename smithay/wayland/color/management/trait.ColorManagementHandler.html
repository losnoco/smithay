fn schedule_image_description_info( &mut self, info: WpImageDescriptionInfoV1, desc: ImageDescription, )

Schedules sending the information events for info (describing desc), to run after the current request dispatch returns — see send_image_description_info. This MUST be deferred (e.g. via an event loop idle callback): wp_image_description_info_v1.done is a destructor event, and destroying the object inside the very callback that created it corrupts wayland-backend’s bookkeeping (it writes the new object’s data after the callback returns, which would then be a use-after-free).

fn image_description_changed(&mut self, _surface: &WlSurface)

Called when a surface’s pending image description changed (set or unset). The committed value becomes visible via get_surface_description after the next wl_surface.commit; compositors typically re-evaluate color handling for the surface’s output on the next redraw.
